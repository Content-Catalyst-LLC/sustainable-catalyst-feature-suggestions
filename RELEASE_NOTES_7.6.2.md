# v7.6.2 — Product Connection Editor

This release adds one governed editor for each canonical product. Administrators can manage product identity, lifecycle, active WordPress implementation, GitHub repository, Release Console presentation, public routes, aliases, validation, and connection history without moving among separate administration screens.

## Added

- Dedicated **Product Connections** administration screen.
- Searchable canonical-product selector.
- Active site and network plugin dropdown.
- GitHub repository and default-branch controls.
- Release Console screen, display order, visibility, and intelligence badges.
- Product, documentation, support, and release-note destinations.
- Governed legacy names, plugin files, slugs, and text-domain aliases.
- Test, synchronize, validate, remove mapping, and reset controls.
- Product-specific connection history.
- Direct **Edit connection** action from Release Operations.

## Compatibility

The canonical product IDs, WordPress option keys, active plugin mappings, GitHub evidence, legacy aliases, public routes, shortcodes, accessibility behavior, and `sustainable-catalyst-feature-suggestions` plugin folder remain preserved.
