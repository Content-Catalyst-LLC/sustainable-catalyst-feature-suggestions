# Connected Help Desk and Support Operations Platform

## Operating model

The connected layer coordinates seven existing operating layers without replacing their records or permission models. Every cross-module command is represented as a plan with an explicit risk class, required authorization, authoritative execution module, and append-only integrity evidence.

## Privacy architecture

Connected dossiers and handoff contexts exclude requester identity, private message bodies, internal notes, attachment contents, access secrets, and secure download URLs. Contact and Engagement remains authoritative for identity, consent, private files, email delivery, secure downloads, and Microsoft Teams scheduling.

## Human authority

The connected layer cannot automatically send customer communication, transition cases, grant access, delete records, create external issues, or deploy software. Those actions remain in specialist modules and require explicit authorization.
