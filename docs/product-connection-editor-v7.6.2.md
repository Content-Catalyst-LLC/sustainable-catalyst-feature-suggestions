# Product Connection Editor — v7.6.2

Open **Support & Feedback → Product Connections** and select a canonical product from the left navigation.

The editor keeps the canonical ID immutable while allowing an administrator to manage:

- public and short names, description, family, lifecycle, release state, and version authority;
- any currently active site or network WordPress plugin;
- GitHub repository, default branch, synchronized version, and sync state;
- Release Console screen, display order, visibility, and intelligence badges;
- product, documentation, support, and release-note routes;
- legacy names, plugin files, folder slugs, and text domains;
- validation state, documentation state, administrative notes, and history.

Use **Save and validate connection** for normal changes. The separate action controls can test or synchronize the GitHub connection, validate the product record, remove the plugin or repository mapping, or reset inherited display values while preserving current connection evidence.
