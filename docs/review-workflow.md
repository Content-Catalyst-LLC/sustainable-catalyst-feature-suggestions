# Review Workflow

Suggested review path:

1. Open **Feature Suggestions** in WordPress admin.
2. Review new private submissions.
3. Classify by category and priority.
4. Decide whether the idea belongs in a module backlog, demo backlog, documentation backlog, Research Librarian backlog, or GitHub issue.
5. Export CSV periodically for offline review.
6. Do not publish user-submitted contact details or private notes.

Possible review statuses:

- New
- Needs review
- Accepted for backlog
- Needs clarification
- Duplicate
- Not planned
- Implemented
