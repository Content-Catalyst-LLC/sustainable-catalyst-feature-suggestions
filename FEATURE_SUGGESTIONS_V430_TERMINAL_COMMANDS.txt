Feature Suggestions v4.3.0 — Repository and Release Synchronization

Place these files in ~/Downloads:

- sustainable-catalyst-feature-suggestions-v4.3.0-repo.zip
  or sustainable-catalyst-feature-suggestions-v4.3.0-release-bundle.zip
- install_and_push_sustainable_catalyst_feature_suggestions_v4_3_0.sh

Run:

cd ~/Downloads

brew install python@3.13
hash -r

SCRIPT="$(ls -t install_and_push_sustainable_catalyst_feature_suggestions_v4_3_0*.sh | head -1)"

chmod +x "$SCRIPT"
bash "$SCRIPT"

For direct WordPress installation, upload:

sustainable-catalyst-feature-suggestions-v4.3.0-wordpress.zip

After activation, open:

Feature Suggestions → Repository Sync

Optional environment configuration:

SCFS_GITHUB_TOKEN
SCFS_GITHUB_WEBHOOK_SECRET

Public repositories do not require a GitHub token. Webhooks remain disabled until a secret is configured and the setting is enabled.
